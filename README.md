# E1-2. Git과 함께하는 Python 첫 발자국

> 이 문서는 `요구사항/Readme.md`를 바탕으로 정리한 작업용 README 초안입니다. 구현이 진행되면 퀴즈 주제, 실행 화면, 실제 파일 구조, Git 이력 정보를 최신 상태로 갱신합니다.

## 프로젝트 개요

이 프로젝트는 Python 3.10 이상과 Git을 사용해 터미널에서 동작하는 나만의 퀴즈 게임을 만드는 미션입니다.

사용자는 메뉴를 통해 퀴즈 풀기, 퀴즈 추가, 퀴즈 목록 확인, 최고 점수 확인, 종료 기능을 사용할 수 있어야 합니다.

프로그램은 프로젝트 루트의 `state.json` 파일에 퀴즈 목록과 최고 점수를 저장하고, 다시 실행해도 데이터를 유지해야 합니다.

또한 기능 단위로 커밋하고, 브랜치를 나누어 작업한 뒤 병합하며, GitHub에 저장소를 공개하는 과정까지 포함합니다.

## 퀴즈 주제와 선정 이유

- 현재 기본 골격의 퀴즈 주제: `Python 기초`
- 선정 이유: 프로그래밍 입문자가 문제를 직접 읽으며 변수, 조건문, 반복문, 자료형, JSON 저장 개념을 함께 복습할 수 있도록 하기 위해 선택했습니다.

이후 필요하면 영화, 게임, 스포츠, 역사 같은 다른 주제로도 쉽게 바꿀 수 있도록 구조를 분리해 두었습니다.

## 개발 목표

- Python 기본 문법을 사용해 입력, 출력, 조건문, 반복문 흐름을 직접 구현합니다.
- `Quiz`, `QuizGame` 같은 클래스로 역할을 분리해 객체 지향 구조를 연습합니다.
- JSON 파일 저장과 불러오기를 통해 데이터 영속성을 구현합니다.
- Git의 `init`, `add`, `commit`, `push`, `pull`, `checkout`, `clone` 흐름을 실습합니다.
- 브랜치 생성과 병합을 경험하며 이후 협업 방식의 기초를 익힙니다.

## 실행 방법

1. 저장소를 클론합니다.
2. 프로젝트 루트로 이동합니다.
3. Python으로 프로그램을 실행합니다.

```bash
git clone <저장소_URL>
cd E1-2
python src/main.py
```

현재 기본 골격은 아래와 같은 메뉴 흐름으로 실행됩니다.

```text
1. 퀴즈 풀기
2. 퀴즈 추가
3. 퀴즈 목록
4. 점수 확인
5. 종료
```

## 기능 목록

### 1. 퀴즈 풀기

- 저장된 퀴즈를 순서대로 또는 확장 기능에 따라 랜덤으로 출제합니다.
- 사용자는 각 문제의 정답 번호를 입력할 수 있습니다.
- 정답 여부를 즉시 안내하고, 모든 문제 풀이 후 최종 점수를 보여줍니다.
- 퀴즈가 없는 경우 안내 메시지를 출력합니다.

### 2. 퀴즈 추가

- 문제, 선택지 4개, 정답 번호를 입력받아 새 퀴즈를 등록합니다.
- 잘못된 입력이 들어오면 다시 입력할 수 있도록 처리합니다.
- 새로 추가한 퀴즈는 `state.json`에 저장합니다.

### 3. 퀴즈 목록 확인

- 현재 저장된 퀴즈 목록을 확인할 수 있습니다.
- 등록된 퀴즈가 없으면 안내 메시지를 출력합니다.

### 4. 점수 확인

- 최고 점수를 확인할 수 있습니다.
- 퀴즈를 풀 때마다 최고 점수를 비교해 갱신합니다.
- 아직 플레이 기록이 없다면 관련 안내 메시지를 출력합니다.

### 5. 저장 및 불러오기

- 프로그램 시작 시 `state.json`을 불러옵니다.
- 파일이 없으면 기본 퀴즈 데이터로 시작합니다.
- 파일이 손상되었거나 읽기 오류가 발생하면 안내 후 복구 또는 초기화합니다.

### 6. 입력 및 예외 처리

- 숫자 입력 전후 공백을 제거하고 처리합니다.
- 빈 입력, 숫자 변환 실패, 허용 범위 밖 숫자를 모두 검증합니다.
- `KeyboardInterrupt`, `EOFError` 발생 시 가능한 범위에서 저장 후 안전하게 종료합니다.

## 데이터 파일 설명

프로그램은 프로젝트 루트의 `state.json` 파일을 사용해 데이터를 저장합니다.

- 경로: `./state.json`
- 인코딩: UTF-8
- 역할: 퀴즈 목록, 최고 점수, 플레이 횟수 영속화

예상 스키마 예시는 아래와 같습니다.

```json
{
  "quizzes": [
    {
      "question": "Python의 창시자는?",
      "choices": ["Guido", "Linus", "Bjarne", "James"],
      "answer": 1
    }
  ],
  "best_score": 3,
  "play_count": 1
}
```

필드 설명:

- `quizzes`: 문제, 선택지, 정답 번호를 담는 퀴즈 목록
- `best_score`: 현재까지 기록된 최고 점수 또는 최고 정답 수
- `play_count`: 사용자가 퀴즈를 완료한 총 횟수

## 클래스 설계 초안

### `Quiz`

- 속성: `question`, `choices`, `answer`
- 역할: 문제를 표현하고 출력하거나 정답 여부를 확인

### `QuizGame`

- 속성: `quizzes`, `best_score`
- 추가 속성: `play_count`, `is_running`
- 역할: 메뉴 표시, 게임 진행, 퀴즈 추가, 목록 출력, 점수 확인, 저장과 불러오기 처리

- 기능 단위로 의미 있는 커밋을 남깁니다.
- 최소 10개 이상의 커밋을 목표로 작업합니다.
